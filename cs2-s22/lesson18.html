CoinGame

Simple coin flipping game for two players. It works as follows:

  1. First, a coin is flipped 9 times.
  2. A human player is then prompted to guess the number of heads.
  3. A computer player guesses a random number of heads (from 0 - 9).
  4. A winner is declared, based on whoever is closer to the real number.

Exercise

  1. Change the game so that the user can enter in the number of times we will flip the coin.
  2. Change the game so that the user can choose who guesses first (the human or the computer player).

(Surprisingly, one of these changes is much harder than the other.)

Exercise

  1. Come up with another strategy for the computer player. Change the game so that the user can select between the two strategies (Random / other strategy).
  2. Change the game to allow for multiple ways to play: Human vs Human, Computer vs Computer, etc. Allowing all possible strategies for computer players as well.

You must complete these changes for homework. It will be due on Thursday 4/7 at 11:59 PM.

Project 3 Discussion

Game:

  • Pick a random number from 3 to 7 for the rows
  • First row: 3 stones.
  • Each row after that: one more than the previous.

Gameplay

  1. Player one picks a row, and takes some stones from that row (only that row).
  2. Player two picks a row, and takes some stones from that row.
  3. Play continues like this until there are no stones left.
  4. The person who takes the last stone wins!

Let’s play it!

Let’s play this on the blackboard?

  • Row 0: * * *
  • Row 1: * * * *
  • Row 2: * * * * *
  • Row 3: * * * * * *

Design Hints

  • Your design should support multiple player types, and all ways to play. Similar to the coin game.
  • Object-oriented concepts will help!
    • If the Game depends on abstract player types, can easily “swap out” different strategies.

Getting Started

  • Make a Player interface
  • Implement a Human Player and a Simple Player.
  • See if you can implement a Game with just these player types.
  • If your Game component relies on the Player interface correctly, you should be able to support four ways of playing:
    • Human vs Simple
    • Simple vs Human
    • Human vs Human
    • Simple vs Simple
