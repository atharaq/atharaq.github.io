Find the mistakes

Problem: given a String name, return the number of times the character c appears in it. What are all the mistakes in the following?

public static int numAppearances(String name, char c) {
    String name = "bob";
    char c = "b";
    int n = 0;
    for (int i = 0; i < name.length(); i++) {
        if (name.charAt(i) == c) {
            System.out.println(c);
            n++;
        }
    }
    System.out.println(n);
}

ArrayList class

  • Built-in class that handles “resizeable arrays”
  • Can use with any type!

Type Parameters

  • Inside angle brackets: the type paramter
  • Must be the name of a class
    • Not a primitive!
    • ArrayList<int> doesn’t work!
  • For primitives: use wrapper classes
    • Integer for int
    • Double for double
    • Character for char

ArrayList Methods

ArrayList<Integer> list = new ArrayList<>();
list.add(1); // list[0] = 1
list.add(5); // list[1] = 5
list.add(10); // list[2] = 10
System.out.println(list.get(1)); // should output 5
System.out.println(list.size()); // should output 3
list.set(2, 27); // list[2] = 27
// list.set(25, 3); throws an exception! list size is only 3!

ArrayList Methods

  • add: add to the “end” of this list
    • add(index, element): add element to list at position \(index\), shift everything over
  • get: get the element at index \(i\)
  • set: change the value of the list at position \(i\)
    • Only works if enough things have been added!
  • remove
  • size

Iterating

// output all the contents of the list

“Foreach” loop syntax above, regular for loop below:

for (int i = 0; i < list.size(); i++) {

Classes

  • ArrayList is a class

  • class: a template for a data type

    ArrayList<Integer> list = new ArrayList<>();
ArrayList<Integer> list2 = new ArrayList<>();

  • list and list2 are different objects of type ArrayList

  • What does new do?

Instantiation

  • The new keyword instantiates the ArrayList object.
  • Instantiation: creating a new object of that type in memory.
  • What does this involve?

Constructors

Another way to define ArrayLists:

ArrayList<String> list = new ArrayList<>(25);
  • ArrayList has multiple constructors
  • Constructor: special method that is called when we the object is created
    • In particular, when the new keyword is used.
  • Used to initialize the data type.

Instance Methods

ArrayList<Integer> list = new ArrayList<>();
ArrayList<Integer> list2 = new ArrayList<>();
list.add(1);
System.out.println("List 1 size: " + list.size());
System.out.println("list 2 size: " + list2.size());

What does this print out? Why?

Instance Methods

  • list and list2 are different objects
    • Keep track of different arrays in memory!
    • The arrays have different elements!
    • The arrays may have different sizes!
  • size is an instance method.
    • as opposed to a static method
  • instance methods: methods which act on specific objects
    • not on the class as a whole!
  • syntax: object.method(parameters)

New data types?

How do we define our own data types?

  • Define a class: public class MyClass
  • In the class:
    • Define a constructor to initialize your data
    • Define instance methods to act on that data
    • How do we specify the data? **instance variables*
  • Outside of the class: use the class!
MyClass m = new MyClass();
m.someMethod();

Data / Behavior

  • Objects encapsulate data and behavior:
  • integer variable:
    • data: actual number it represents
    • behavior: add, subtract, etc.
  • ArrayList variable?
    • data?
    • behavior?
  • User-defined types? often:
    • data: instance variables
    • behavior: instance methods
