Put it together

One type of problem I didn’t put on the review:

  • You’ll be given a problem broken into components
  • Write each component in its own method
  • Write a main method that uses all the components

Example

Write a program that asks the user for the current time (hour and minute, using a 24 hour clock), and a number of minutes, and outputs the time after that many minutes.

You’ll first implement the following methods:

getNumber

public static int getNumber(Scanner scanner, String prompt, int lowerBound, int upperBound, String error)
  • This method should display the “prompt” message, get input from the scanner, and return an integer between lowerBound and upperBound, inclusive.
  • For any invalid input, output the error message and try again.

displayTime

public static String displayTime(int hour, int minute)
  • This method returns a String version of the time
  • Should handle cases like the minute being single or double digits

hourAfter and minuteAfter

public static int hourAfter(int oldHour, int oldMinute, int minutesToAdd) {

}
public static int minuteAfter(int oldHour, int oldMinute, int minutesToAdd) {

}

These methods do the actual computation of the new hour and minute

main

public static void main(String[] args) {

}
  • After implementing all of the above methods, use them to implement the main program.
